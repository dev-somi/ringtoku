프로젝트 개요

🎯

Problem Statement

일본인 영어 학습자는 발화 기회가 부족하고, 기존 AI 챗봇은 매 대화가 리셋되어 개인화가 없다

💡

Our Solution

매일 AI가 전화를 걸어주고, 과거 대화를 기억해 개인화된 영어 회화 및 문법 교정을 제공

🚀

Core Differentiator

RAG 기반 장기 기억: "저번에 말씀하신 고양이 Leon은 잘 있나요?" 수준의 기억력

📊

Market Fit

일본 영어 교육 시장 2.3조엔 규모, AI 튜터 수요 급성장 중

서비스 한 줄 정의

📌
"RingToku는 RAG 기반 장기 기억 기술을 활용하여, 어제의 대화를 내일의 복습으로 연결하는 일본 맞춤형 AI 영어 회화 Flutter 앱입니다."

타겟 사용자 & 페르소나

👩‍💼

Primary: 직장인 Yuki (28세)

영어 회의가 두렵고, 매일 짧은 시간 연습하고 싶다. 발음보다 "자연스러운 대화"가 목표

🎓

Secondary: 대학생 Kenji (22세)

해외 취업 목표. TOEIC은 있는데 실전 회화 경험이 없다. 포트폴리오로 영어 이력 쌓고 싶음

👨‍💻

Tertiary: 엔지니어 Hiroshi (35세)

해외 컨퍼런스 참가를 위해 비즈니스 영어 필요. 체계적인 문법 교정 및 분석 리포트를 원함

User Stories

As Yuki, I want to receive a daily call so that I practice speaking even on busy days without planning ahead.
As Kenji, I want the AI to remember my goals and past topics so that each conversation feels personally tailored.
As Hiroshi, I want to see a grammar error report after each call so that I can track my improvement over time.

핵심 기능 정의

기능 설명 우선순위 단계
📞 Daily AI Call 매일 지정 시간에 앱 푸시 알림 → 탭하면 AI와 음성 통화 시작 (10~15분) P0 MVP
🧠 Memory Recall 과거 대화 내용을 Vector DB에 저장, 새 통화 시작 시 관련 기억을 검색해 자연스럽게 언급 P0 MVP
📝 Grammar Error Log 통화 중 감지된 문법 오류를 실시간 분석, 통화 종료 후 리포트로 제공 P0 MVP
📊 Progress Dashboard 주간 통화 횟수, 총 통화 시간, 자주 틀리는 문법 패턴 시각화 P1 MVP
🔁 Smart Recall Review 에빙하우스 망각곡선 기반으로 3일 전 틀렸던 표현을 오늘 대화에 자연스럽게 재등장 P1 V2
🇯🇵 JP Assist Mode 영어가 막힐 때 일본어로 질문하면 바로 설명해주는 일본어 보조 모드 P1 V2
💼 Business / Casual 모드 격식 있는 비즈니스 영어와 일상 대화를 구분, 모드별 다른 AI 페르소나 적용 P1 V2
🏆 Streak & 배지 연속 통화 일수, 총 회화 시간 기반 업적 시스템으로 지속 학습 유도 P1 V3

Memory Recall 동작 흐름 예시

💬
1일차: "I have a cat named Leon."
→ AI가 요약: user_has_cat: Leon으로 Vector DB 저장

4일차: "I'm so tired today."
→ DB 검색 → Leon 관련 기억 매칭
→ AI 응답: "I'm sorry to hear that. Maybe spending some time with Leon would help?"

기술 아키텍처

기술 스택

Frontend
Flutter 3.x — iOS / Android 크로스플랫폼, 전화 UI 구현
STT
OpenAI Whisper API — 실시간 음성 → 텍스트 변환 (다국어 지원)
LLM
Gemini 1.5 Flash (실시간 대화) + Gemini 1.5 Pro (문법 분석 후처리)
TTS
Google Cloud TTS 또는 ElevenLabs API — 자연스러운 영어 음성 합성
Vector DB
Supabase pgvector 또는 Pinecone — 대화 요약 임베딩 저장 및 유사도 검색
Backend
Python FastAPI — STT/LLM/TTS 오케스트레이션, RAG 파이프라인
DB (RDB)
Supabase PostgreSQL — 사용자 정보, 통화 이력, 문법 오류 로그
Auth
Supabase Auth — 이메일 / 구글 로그인
Push 알림
Firebase Cloud Messaging (FCM) — 매일 통화 알림
로컬 DB
Drift (Flutter) — 오프라인 캐시, 통화 기록 로컬 저장

RAG 파이프라인 흐름

[통화 종료]
  ↓
Whisper STT → 전체 transcript 생성
  ↓
LLM → 대화 요약 + 개인 정보 추출 (이름, 관심사, 목표 등)
  ↓
OpenAI Embeddings API → 벡터 생성
  ↓
Vector DB (Supabase pgvector) 저장

[새 통화 시작]
  ↓
"오늘 기분 어때요?" → 임베딩 생성 → Vector DB 유사도 검색
  ↓
관련 과거 대화 컨텍스트 LLM에 주입 (RAG)
  ↓
AI가 기억을 활용해 자연스러운 대화 시작

데이터 모델

테이블주요 필드용도
users id, email, name, learning_goal, call_time, streak_count, created_at 사용자 기본 정보
calls id, user_id, started_at, ended_at, duration_sec, transcript_text, summary 통화 이력 전체 저장
grammar_errors id, call_id, user_id, original_text, corrected_text, error_type, occurred_at 문법 오류 로그 & 분석
memories id, user_id, content, embedding (vector), source_call_id, created_at RAG용 Vector DB 핵심 테이블
review_queue id, user_id, grammar_error_id, next_review_at, reviewed_count 에빙하우스 망각곡선 복습 스케줄

개발 로드맵

Week 1–2 · MVP Phase 1

기반 인프라 구축

Flutter 프로젝트 셋업, Supabase 연동, FastAPI 백엔드 기본 구조, Auth 구현

Week 3–4 · MVP Phase 2

음성 통화 핵심 기능

STT (Whisper) → LLM (Gemini) → TTS 파이프라인 구현, Call Screen UI, 실시간 자막

Week 5–6 · MVP Phase 3

RAG 메모리 시스템

대화 요약 & 임베딩 저장, Vector 검색 파이프라인, Memory Tab UI, 기억 기반 대화 테스트

Week 7–8 · MVP Phase 4

문법 분석 & 리포트

Grammar Error 감지 로직, Report Screen UI, 진행 대시보드, FCM 푸시 알림

Week 9–10 · V2

고도화 & 일본 시장 특화

JP Assist Mode, Business/Casual 모드, 에빙하우스 복습 큐, 스트릭 & 배지 시스템

Week 11–12 · Launch Prep

QA & 출시 준비

버그 수정, 성능 최적화, TestFlight/Play Store 내부 테스트, 랜딩 페이지 제작

🎤
"기존 AI 영어 학습 앱들은 대화가 종료되면 모든 컨텍스트가 초기화됩니다. RingToku는 RAG(Retrieval Augmented Generation) 기술을 활용해 사용자의 모든 발화 데이터를 임베딩 벡터로 변환, Vector DB에 저장합니다. 다음 통화 시 이 데이터를 유사도 검색으로 소환해 AI가 마치 오랜 친구처럼 사용자를 기억합니다. 이를 통해 학습 지속률을 높이고 개인 맞춤형 문법 교정을 제공합니다."
